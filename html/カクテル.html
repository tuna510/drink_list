カラム全部選択したい場合はアナル使う(SELECT *…みたいな)
全はアナル、アナルは全。つまりヨグ=ソトース、はっきりわかんだね。


■SELECT文　データの取得
SELECT カラムA,カラムB,… FROM テーブル名 WHERE 条件文;
例 
SELECT * FROM user WHERE gender="male";
NULLが入っている部分のデータを取得したい場合は
SELECT カラムA,カラムB,… FROM テーブル名 WHERE 該当のカラム IS NULL;
と記述する。WHERE 該当のカラム=NULLだと取れない。
NULLが要らない場合はWHERE 該当のカラム IS NOT NULL;にすればOK


■INSERT文　データの追加
INSERT INTO テーブル名(カラムA,カラムB,…) VALUES('カラムAに入れるデータ','カラムBに云々',…);
例 
INSERT INTO user(name,age,gender) VALUES('Genya',16,'male');
-- テキスト型(上記だとname,gender)に入れたい情報は''で囲み、数字は囲まずにそのまま(age)


■UPDATE文　データの更新
UPDATE テーブル名 SET 更新処理 WHERE 条件文;
例 
UPDATE user SET name="Sumiko",gender="female?" WHERE id=1;


■DELETE文　データの削除
DELETE FROM テーブル名 WHERE 条件文;
例 
DELETE FROM user WHERE id=1;
-- WHEREで消す部分を指定しないと全部消えてしまうので注意！



■AND演算子　複数の条件を指定したい
SELECT カラムA,カラムB,… FROM テーブル名 
WHERE 条件文1
AND 条件文2;
例 
SELECT * FROM user 
WHERE age=16
AND gender="male";


■OR演算子　条件のどれかに該当するやつ全部選択したい
SELECT カラムA,カラムB,… FROM テーブル名 
WHERE 条件文1
OR 条件文2;
例 
SELECT * FROM user 
WHERE age=16
OR gender="male";



■ソート　ORDER BY
SELECT カラムA,カラムB,… FROM テーブル名 ORDER BY ソートしたいカラム 順番の指定;
-- 順番の指定……ASC(昇順)、DESC(降順)
例 
SELECT * FROM user ORDER BY age ASC;



■レコードの数指定　LIMIT
SELECT カラムA,カラムB,… FROM テーブル名 LIMIT 取り出したい数;
-- ORDER BYでソートしてからやると良い感じに。LIMITつける位置は一番最後
-- WHEREとも併用できるよ
例 
SELECT * FROM user ORDER BY id DESC LIMIT 3;



■テーブルの結合　がっちゃんこ
内部結合
SELECT カラムA, カラムB,… FROM テーブル名A
    INNER JOIN テーブル名B ON 結合の条件;
    -- 内部結合の場合、ベースとなるテーブルから、条件にマッチするレコードがないものは削除される
外部結合
SELECT カラムA, カラムB,… FROM テーブル名A
    -- LEFTは左側のテーブルを、RIGHTは右側のテーブルを軸にして結合
    LEFT(RIGHT) OUTER JOIN テーブル名B ON 結合の条件;

            ■分岐　CASE

            

                
SELECT カラム CASE // 「CASE」と「END」で囲んだ中に条件を記載
    WHEN 条件 THEN 条件を満たしたら表示する内容
    // (他にも分岐させたい内容があればまたWHEN…で)
    ELSE 条件を満たしていなかったら表示する内容
    END
FROM テーブル名;

            ■曖昧検索　LIKE　%がワイルドカード(どんな文字列にも一致することを指す記号)になる

            

                
SELECT カラムA,カラムB,… FROM テーブル名 WHERE 検索に使うカラム LIKE 曖昧検索の条件;
// 例えば前に「山」がつく名前を探したいなら"山%"、後ろにつく名前なら"%山",
// 山がつくもの全てなら"%山%"と書いておく。曖昧部分が１文字だけなら%の代わりに_を使う。
// また、条件に一致しない物を探したいならLIKEではなくNOT LIKEを使う。
// それと検索条件に「％」とか「_」を入れたい場合は「\_」や「\%」にすればOK(エスケープ文字)

            

            大文字小文字の区別つけたい
            

                
SELECT カラムA,カラムB,… FROM テーブル名 WHERE 検索に使うカラム LIKE BINARY 曖昧検索の条件;
